# GitHub Pages 部署指南

## 方法一：通过 GitHub 网页操作（最简单）

### 1. 创建仓库
1. 访问 https://github.com/new
2. 仓库名称：`notebook-app`（或其他名称）
3. 选择 Public（公开）
4. 点击 "Create repository"

### 2. 上传文件
1. 在新建的仓库页面，点击 "uploading an existing file"
2. 将 `Kimi_Agent_Deployment_v1` 文件夹内的所有文件拖拽到页面
3. 等待上传完成
4. 填写提交信息：`Initial commit`
5. 点击 "Commit changes"

### 3. 启用 GitHub Pages
1. 进入仓库的 Settings（设置）
2. 左侧菜单找到 "Pages"
3. Source 选择 "Deploy from a branch"
4. Branch 选择 "main"，文件夹选择 "/ (root)"
5. 点击 "Save"
6. 等待几分钟，页面会显示访问地址

### 4. 访问网站
访问地址：`https://chiyintao.github.io/notebook-app/`

## 方法二：使用 Git 命令行（推荐）

### 1. 初始化本地仓库
```bash
cd Kimi_Agent_Deployment_v1
git init
git add .
git commit -m "Initial commit"
```

### 2. 关联远程仓库
```bash
# 在 GitHub 创建仓库后，复制仓库地址
git remote add origin https://github.com/chiyintao/notebook-app.git
git branch -M main
git push -u origin main
```

### 3. 启用 GitHub Pages
按照方法一的第3步操作

---

## 方法三：使用 GitHub Desktop（图形界面）

### 1. 安装 GitHub Desktop
下载：https://desktop.github.com/

### 2. 创建仓库
1. 打开 GitHub Desktop
2. File → New Repository
3. 名称：`notebook-app`
4. Local Path：选择 `Kimi_Agent_Deployment_v1` 的父文件夹
5. 点击 "Create Repository"

### 3. 发布到 GitHub
1. 点击 "Publish repository"
2. 确认仓库名称
3. 取消勾选 "Keep this code private"（保持公开）
4. 点击 "Publish Repository"

### 4. 启用 GitHub Pages
按照方法一的第3步操作

## 自定义域名（可选）

如果你有自己的域名：

### 1. 在仓库根目录创建 CNAME 文件
```bash
echo "your-domain.com" > CNAME
git add CNAME
git commit -m "Add custom domain"
git push
```

### 2. 配置 DNS
在域名服务商添加 CNAME 记录：
```
类型: CNAME
主机记录: www
记录值: chiyintao.github.io
```

### 3. 在 GitHub Pages 设置中填写域名
Settings → Pages → Custom domain → 填写域名 → Save

## 常见问题

### Q: 网站显示 404
A: 等待几分钟，GitHub Pages 需要时间构建。检查 Settings → Pages 是否显示绿色的访问地址。

### Q: 样式不正常
A: 检查文件路径是否正确，确保所有 CSS 和 JS 文件都已上传。

### Q: 如何查看访问统计
A: 可以使用 Google Analytics 或其他统计工具。

### Q: 数据会丢失吗
A: 数据存储在用户浏览器的 localStorage，清除浏览器数据会丢失。建议定期使用导出功能备份。

## 进阶：自动部署

创建 `.github/workflows/deploy.yml`：

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./
```

这样每次推送代码都会自动部署。
